## Template Rules

1. **Executive summary first** -- reader should understand impact without scrolling
2. **Group scenarios by feature** -- not by file or change type
3. **Embed screenshots inline** -- immediately after the scenario steps they document
4. **Coverage matrix is mandatory** -- every changed file must appear, even if not covered
5. **Recommendations section** -- always include at least one actionable suggestion
6. **No empty sections** -- if a section has no content, remove it rather than leaving a placeholder
