# CSV Format Reference (e2e-test Compatible)

CSV files generated by this skill must be directly executable by the `e2e-test` skill. Follow this format exactly.

## Structure

```csv
# config
url,{base-url}
viewport,{width}x{height}
setup,{precondition actions}
timeout,{seconds}

# steps
step,action,target,input,expected_result
1,navigate,{url},,{expected page state}
2,{action},{human-readable target},{input},{expected result}
```

## Config Section

| Key | Required | Example | Notes |
|-----|----------|---------|-------|
| `url` | Yes | `http://localhost:3000` | Base URL of the app |
| `viewport` | No | `1280x720` | Default: 1280x720 |
| `setup` | No | `Clear localStorage` | Executed before steps |
| `timeout` | No | `10` | Seconds per step, default: 10 |

## Action Vocabulary

These actions map to Playwright MCP tools in the e2e-test skill:

| Action | Tool | Description |
|--------|------|-------------|
| `navigate` | `browser_navigate` | Go to a URL |
| `click` | `browser_click` | Click an element (by ref from snapshot) |
| `type` | `browser_type` | Type text into a focused element |
| `fill` | `browser_fill_form` | Fill a form field by label |
| `hover` | `browser_hover` | Hover over an element |
| `select` | `browser_select_option` | Select a dropdown option |
| `keypress` | `browser_press_key` | Press a keyboard key |
| `dblclick` | `browser_click` | Double-click an element |
| `wait` | `browser_wait_for` | Wait for an element/condition |
| `screenshot` | `browser_take_screenshot` | Capture current page state |
| `verify` | `browser_snapshot` | Verify element state via accessibility tree |
| `assert_text` | `browser_snapshot` | Assert specific text is present |
| `assert_visible` | `browser_snapshot` | Assert element is visible |
| `assert_hidden` | `browser_snapshot` | Assert element is not visible |
| `assert_url` | `browser_evaluate` | Assert current URL matches |
| `dialog` | `browser_handle_dialog` | Handle alert/confirm/prompt |
| `upload` | `browser_file_upload` | Upload a file |
| `evaluate` | `browser_evaluate` | Run JavaScript in page context |

## Target Conventions

Targets must be **human-readable descriptions** matched against the accessibility tree. Do NOT use CSS selectors.

### Good targets
- `Submit button`
- `Email input field`
- `Navigation menu`
- `"Add to Cart" button`
- `Username text field`
- `Main heading`
- `Toggle checkbox for "Remember me"`

### Bad targets (do NOT use)
- `#submit-btn`
- `.form-input.email`
- `div > form > button:first-child`
- `[data-testid="submit"]`

## File Naming

`scenario-NNN-{feature-slug}.csv`

Examples:
- `scenario-001-contact-form.csv`
- `scenario-002-user-login.csv`
- `scenario-003-product-search.csv`

## CSV Rules

1. One CSV per logical test scenario (one user flow)
2. Steps are executed sequentially in order
3. Each step must have a clear expected_result
4. Use `screenshot` action to capture visual evidence at key points
5. Commas in field values must be quoted: `"value with, comma"`
6. Empty fields use empty string (not null or N/A)
7. Keep step count reasonable (5-15 steps per scenario)
